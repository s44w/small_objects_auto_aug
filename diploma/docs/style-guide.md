## 1. Конвенции при написании markdown

### 1.1 Язык и стиль

- **Только русский язык**, безличный научный стиль: *"в работе предложено"*, *"получено"*, *"проведён эксперимент"*. Не "я сделал", не "мы решили".
- Никакого разговорного регистра, англицизмов без необходимости. Заимствованные термины (микросервис, фреймворк, латентность) — допустимы, см. `glossary.md`.
- **Каждое утверждение** в основной части — со ссылкой на источник `[N]` или на собственную разработку (код model-forge, эксперимент в главе "Экспериментальное исследование"). Никаких "по нашим данным" без референса.

### 1.2 Структура работы (порядок разделов)

Точно как у Маршуниной + НИР 7 семестра:

1. Титульный лист (без номера страницы)
2. РЕФЕРАТ
3. СОДЕРЖАНИЕ
4. (Опционально) Перечень условных обозначений и сокращений
5. ВВЕДЕНИЕ
6. Основные главы (несколько, **без цифровой нумерации в названии**)
7. ЗАКЛЮЧЕНИЕ
8. СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ
9. ПРИЛОЖЕНИЕ А, ПРИЛОЖЕНИЕ Б, ... (если есть)

### 1.3 Заголовки и регистр (КРИТИЧНО)

**Главное правило:** в **тексте** (в markdown и в финальном docx) **нет цифровой нумерации** в заголовках глав и параграфов. Всё нумерация — автоматически через стили Word `+Заголовок 1/2/3 уровня` (multilevel list внутри стиля).

**Регистр:**
- **Спецразделы** (`Реферат`, `Содержание`, `Введение`, `Заключение`, `Список использованных источников`, `Приложение А`, `Приложение Б`) в markdown пишутся **в Title Case** (`# Введение`, не `# ВВЕДЕНИЕ`). ПРОПИСНЫМИ они отрисовываются в финальном docx автоматически — через character-level `w:caps` в Word-стилях `+ЗАГОЛОВОК по центру` и `+ЗаголРеферСодерж`. Это нужно, чтобы в TOC раздел появлялся как `Введение`, а не `ВВЕДЕНИЕ` (TOC извлекает raw текст).
- **Title Case (с большой буквы первое слово, остальное обычно)** — для заголовков глав и параграфов: `Аналитический обзор методов 3D-реконструкции`, `Постановка задачи 3D-реконструкции`.

**Иерархия в markdown:**

| Уровень | Markdown | Пример | Стиль Word при компиляции |
|---|---|---|---|
| Спецразделы (РЕФЕРАТ, ВВЕДЕНИЕ, ЗАКЛЮЧЕНИЕ, СПИСОК...) | `# ВВЕДЕНИЕ` | `# ВВЕДЕНИЕ` | `+ЗАГОЛОВОК по центру` |
| Глава (нумеруется автоматом) | `# Название главы` | `# Аналитический обзор методов 3D-реконструкции` | `+Заголовок 1 уровня` |
| Параграф | `## Название параграфа` | `## Постановка задачи 3D-реконструкции` | `+Заголовок 2 уровня` |
| Подпараграф | `### Название подпараграфа` | `### GAN-подходы для 3D-генерации` | `+Заголовок 3 уровня` |

Без точки в конце заголовков.

### 1.4 Ссылки на литературу

**Формат в тексте:** квадратные скобки с номером источника из `sources.md`.
- Одиночная: `[5]`
- Со страницей (буквальная цитата): `[5, с. 23]` или `[5, p. 142]` для англоязычных
- Множественные: `[3, 5, 7]` или `[3–5]` (диапазон)

**Как в примерах:** в ВКР Маршуниной и в предыдущих работах ссылки в основном тексте оформлены именно как числовые ссылки в квадратных скобках, например `[1]`, `[3]`, `[5]`. Полные библиографические записи и URL-адреса не вставляются внутрь основного абзаца, а приводятся в списке использованных источников и в рабочих списках источников раздела.

**Примеры:**
> Метод нейронных излучательных полей был предложен в работе [4]. Альтернативный подход через диффузионные модели описан в [7–9].

В `sources.md` — **в порядке упоминания** в тексте (не алфавит). При написании секции — каждый новый источник добавляется со следующим номером.

**Обязательное правило для электронных источников:** в `sources.md`, в конце тематического `.md`-файла и в итоговом списке источников для каждого внешнего электронного ресурса должен быть указан полный URL в виде `https://...`. URL не должен быть сокращённым, не должен быть повреждён, должен соответствовать реальному источнику и открываться.

### 1.5 Перечни (списки)

Из `source-materials/formatting-rules/правила_оформления_перечней.pdf`:

**Базовая форма** — перед каждым элементом ставится **дефис**:
```
Платформа предоставляет следующие возможности:
— загрузка изображения;
— асинхронная обработка;
— скачивание модели.
```

**С буквенными метками:**
```
а) первый элемент;
б) второй элемент;
в) третий элемент.
```
(Строчные русского алфавита, исключая: й, ё, з, о, ч, ь, ы, ъ.)

**С цифровыми метками** (для вложенных уровней):
```
1) первый;
2) второй;
3) третий.
```

**Знаки препинания в конце:**
- **Запятая** или **точка с запятой** — простые элементы со строчной буквы.
- **Точка** — если элементы с прописной (полные предложения).

В markdown — записывать как обычный список (`- пункт` для маркированных, `1. пункт` для нумерованных). При компиляции `scripts/apply-stosgau-styles.lua` разворачивает `BulletList`/`OrderedList` в плоскую последовательность абзацев со стилем `+Абзац с отступом 1-ой строки`, префиксуя каждый пункт en-dash `–` (для маркированных) или `N)` (для нумерованных). Никаких маркеров-кружочков и автонумерации Word — это критично, иначе СТО СГАУ нарушен.

### 1.6 Рисунки

**В markdown:**
```markdown
![](../assets/diagrams/architecture.png)

Рисунок 1 – Архитектура микросервисной системы
```

**Правила:**
- Подпись **под** рисунком, выравнивание по центру (стиль `+№ - Название рисунка`).
- Формат: `Рисунок N – Название` где **`–` это короткое тире (en-dash, U+2013), НЕ длинное `—`**.
- Нумерация **сквозная по работе** (как у Маршуниной): Рисунок 1, Рисунок 2, ... (а не 1.1, 1.2).
- Ссылка в тексте — обязательна **до появления рисунка**: *"Архитектура показана на рисунке 1"* или *"(см. рисунок 1)"*.
- Если рисунок взят из источника: `Рисунок 5 – Архитектура NeRF [4]`.
- **Alt-text у изображений всегда пустой** (`![](path)`, не `![описание](path)`). Иначе Pandoc автоматически рендерит figure-caption из alt-text и под рисунком появляется дубль помимо нашей правильной подписи `Рисунок N – Название`. Проверка: `grep '!\[[^]]' diploma/text/*.md` должно возвращать 0 совпадений.

### 1.7 Таблицы

**В markdown:**
```markdown
Таблица 1 – Сравнение производительности микросервисов

| Метрика | REST | gRPC | Kafka |
|---|---|---|---|
| Латентность p50 | 50 мс | 15 мс | 30 мс |
```

**Правила:**
- Подпись **над** таблицей, выравнивание слева, без абзацного отступа (стиль `+№ - Название таблицы`).
- Формат: `Таблица N – Название` (en-dash `–`).
- Нумерация **сквозная по работе**.
- Если таблица переносится — на след. странице: `Продолжение таблицы 1` (без названия), последняя часть: `Окончание таблицы 1`.
- Ссылка в тексте — обязательна: *"Результаты приведены в таблице 1"*.

### 1.8 Формулы

**В markdown:**
```markdown
Расстояние Чамфера между двумя точечными облаками вычисляется по формуле:

$$
d_{CD}(P, Q) = \sum_{p \in P} \min_{q \in Q} \|p - q\|^2 + \sum_{q \in Q} \min_{p \in P} \|p - q\|^2
$$ (1)

где $P$ и $Q$ — точечные облака, $\|\cdot\|$ — евклидова норма.
```

- Формула — на отдельной строке, по центру.
- Номер — справа в круглых скобках: `(1)`, `(2)`, ... сквозная нумерация.
- Ссылка в тексте: *"по формуле (1)"*.
- Объяснение переменных — со слова "где".

### 1.9 Сокращения

- Первое упоминание — полное название, в скобках сокращение: *"глубокое обучение (deep learning, DL)"*.
- Дальше — сокращение.
- Если сокращений >5 — отдельный раздел "Перечень условных обозначений и сокращений".

### 1.10 Тире и пунктуация

- **Короткое тире `–` (en-dash, U+2013)** – единственное тире, которое используется в работе. Применяется во всех случаях: в пояснительных конструкциях (*"микросервис – это..."*), в подписях рисунков/таблиц (`Рисунок 1 – Название`), в ссылках на литературу-разделителях (`М. : Наука, 1977. – 656 с.`), в диапазонах (`5–10 эпох`, `[3–5]`).
- **Длинное тире `—` (em-dash, U+2014) – не использовать.** Замена производится на en-dash во всём тексте автоматическим bulk-replace; проверка – `grep '—' diploma/text/*.md` должно возвращать 0 совпадений.
- **Дефис `-`** – в составных словах: *"научно-техническая"*.
- **Кавычки** – «ёлочки» для русского, "лапки" для англоязычных названий.
- Между числом и единицей – **неразрывный пробел**: `12 МБ`, `30 мм`.

## 2. Список использованных источников (формат ГОСТ 7.1-2003)

### 2.1 Общие принципы

- Нумерация **в порядке упоминания** в тексте.
- Между элементами описания — **точка-тире** (` . – `), где тире — короткое (en-dash).
- Стиль Word: `+Список использованных источников`.

### 2.2 Книга (1–3 автора)

```
Самарский, А.А. Теория разностных схем [Текст] : учеб. пособие / А.А. Самарский.
– М. : Наука, 1977. – 656 с.
```

```
Тихонов, А.Н. Уравнения математической физики [Текст] : учеб. пособие / А.Н. Тихонов,
А.А. Самарский. – М. : Наука, 1972. – 736 с.
```

### 2.3 Книга (4 и более авторов)

```
Методы компьютерной оптики [Текст] : учеб. для вузов / А.В. Волков [и др.] ;
под ред. В.А. Сойфера. – 2-е изд., испр. – М. : Физматлит, 2003. – 688 с.
```

### 2.4 Статья из журнала

```
Иванов, А.А. Метод оптимизации нейронных сетей [Текст] / А.А. Иванов, Б.Б. Петров //
Известия РАН. Информационные процессы. – М., 2020. – Т. 15, № 3. – С. 45–58.
```

### 2.5 Электронный ресурс

```
Spring Boot Reference Documentation [Электронный ресурс]. – Электрон. дан. –
VMware Tanzu, 2024. – Режим доступа : https://docs.spring.io/spring-boot/docs/current/reference/html/
(дата обращения: 26.04.2026).
```

```
Tochilkin, D. TripoSR: Fast 3D Object Reconstruction from a Single Image [Electronic resource] /
D. Tochilkin, [et al.]. – arXiv preprint arXiv:2403.02151, 2024. –
URL : https://arxiv.org/abs/2403.02151 (accessed: 26.04.2026).
```

Для всех электронных источников URL обязателен. При добавлении новых интернет-источников необходимо:

- вставлять полный URL;
- проверять, что URL ведёт именно на нужную статью, документацию или датасет;
- не заменять URL на укороченные или промежуточные ссылки;
- по возможности использовать первичный источник, а не пересказ или агрегатор.

## 3. Стили Word из шаблона (для финальной компиляции)

При сборке через `pandoc text/*.md -o final.docx --reference-doc=template/blank-template.docx` markdown-элементы маппятся на эти стили:

| Markdown | Стиль Word |
|---|---|
| `# ВВЕДЕНИЕ` (спецраздел) | `+ЗАГОЛОВОК по центру` |
| `# Глава` | `+Заголовок 1 уровня` |
| `## Параграф` | `+Заголовок 2 уровня` |
| `### Подпараграф` | `+Заголовок 3 уровня` |
| Обычный абзац | `+Абзац с отступом 1-ой строки` |
| Подпись рисунка | `+№ - Название рисунка` |
| Подпись таблицы | `+№ - Название таблицы` |
| Запись в списке литературы | `+Список использованных источников` |
| Список / перечень | `List Paragraph` |
| Заголовок РЕФЕРАТ/СОДЕРЖАНИЕ | `+ЗаголРеферСодерж` |
| Ключевые слова реферата | `+КлючСловаРеферат` |
| Заголовок приложения | `+ПРИЛОЖЕНИЕ А и Загол. на сл. строке` |
| Элементы титула | `+Тит_Абзац по центру`, `+Тит_Тема ВКР` |

## 4. Типографское оформление (применяется автоматически из шаблона)

Параметры извлечены из `ВКРБ_МаршунинаМА.docx` и подтверждены НИР 7 семестра.

### 4.1 Поля страницы

- Левое: **30 мм**
- Правое: **15 мм**
- Верхнее: **20 мм**
- Нижнее: **20 мм**

### 4.2 Шрифт и абзац (внутри стилей шаблона)

- Шрифт: **Times New Roman**, размер **14 pt** (для основного текста).
- Цвет: чёрный.
- Межстрочный интервал: **полуторный (1.5)**.
- Абзацный отступ: **1.25 см** (стиль `+Абзац с отступом 1-ой строки`).
- Выравнивание: **по ширине**.
- Перенос слов: автоматический.

### 4.3 Нумерация страниц

- Сквозная, **арабскими цифрами**.
- На титульном листе номер не ставится, но засчитывается.
- Расположение — по шаблону Маршуниной (внизу).

### 4.4 Объём

- ВКР бакалавра: **50–70 страниц основного текста** (без титула, реферата, приложений).
- Целевой объём для нашей работы: **60–70 стр.**

### 4.5 Структура титульного листа

(По примеру Маршуниной — точно эта последовательность):
1. МИНИСТЕРСТВО НАУКИ И ВЫСШЕГО ОБРАЗОВАНИЯ РОССИЙСКОЙ ФЕДЕРАЦИИ
2. федеральное государственное автономное образовательное учреждение высшего образования «Самарский национальный исследовательский университет имени академика С.П. Королёва»
3. **Институт информатики и кибернетики**
4. **Кафедра технической кибернетики**
5. ВЫПУСКНАЯ КВАЛИФИКАЦИОННАЯ РАБОТА
6. Тема ВКР (стиль `+Тит_Тема ВКР`)
7. по программе бакалавриата по направлению подготовки 01.03.02 Прикладная математика и информатика, профиль «Компьютерные науки» (или твой профиль)
8. Обучающийся: М.И. Жиляев + подпись
9. Научный руководитель ВКР, к.ф.-м.н., доцент: А.А. Белоусов + подпись
10. **Нормоконтролёр**: ФИО + подпись
11. Самара, 2026 (внизу)

## 5. Чек-лист перед закрытием задачи в beads

- [ ] Все утверждения имеют ссылку на источник `[N]` или код/собственный эксперимент.
- [ ] Заголовки **без цифровой нумерации** в тексте (нумерация автоматом из стилей).
- [ ] Спецразделы в markdown — в Title Case (`# Введение`, `# Заключение`); ПРОПИСНЫМИ их сделает Word-стиль автоматически.
- [ ] Подписи рисунков/таблиц с **коротким тире `–`** (en-dash, не em-dash).
- [ ] Иерархия `#` / `##` / `###` соответствует пункту 1.3.
- [ ] Между числом и единицей — неразрывный пробел.
- [ ] Списки оформлены по 1.5 (дефис, или а)/б), или 1)/2)).
- [ ] Используется только согласованная терминология из glossary.md.
- [ ] Первое упоминание сокращения — с расшифровкой.
- [ ] Никаких "я", "мы решили", "по нашим данным".
- [ ] Источник, на который ссылаешься, добавлен в sources.md в формате ГОСТ 7.1-2003.
- [ ] Для каждого электронного источника указан полный и корректный URL.
