# Инструкции для работы с текстом ВКР

Этот каталог используется как рабочая зона для написания ВКР по проекту
адаптивного подбора аугментаций для детекции малых объектов.

## Основные правила

Основные правила:

1. Опираться только на реальный код и документы текущего репозитория.
2. Использовать русский научный стиль без "я" и "мы".
3. Все утверждения о реализации подтверждать ссылками на файлы проекта.
4. Все утверждения о литературе подтверждать ссылками из `docs/sources.md`.
5. Заголовки в Markdown писать без ручной нумерации.

## Куда смотреть по содержанию

Куда смотреть по содержанию:

- `src/pipeline_mvp.py` - общий конвейер.
- `src/data/` - подготовка и валидация датасетов, tiling, fixtures.
- `src/analysis/` - аналитика датасета.
- `src/policy/` - rule engine и схема policy.
- `src/augmentation/` - кастомные трансформации и object bank.
- `src/training/` - сценарии обучения.
- `src/evaluation/` - COCO-конвертация, COCOeval, inference.
- `src/experiments/` - AutoAug-like comparison и агрегирование результатов.
- `docs/` - уже существующие документы проекта.
- `tests/` - минимальная проверяемая база для описания корректности модулей.

## Порядок работы

Порядок работы:

1. Сначала уточнить структуру главы в `docs/narrative.md`.
2. Затем проверить терминологию в `docs/glossary.md`.
3. После этого писать текст в `text/*.md`.
4. Перед сборкой проверить пути к рисункам и таблицам.
